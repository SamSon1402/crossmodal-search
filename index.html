Finds infringing listings semantically — even when counterfeiters remove brand names from titles

Problem this solves

Sophisticated counterfeiters strip brand names from listing titles ("premium athletic runner" instead of "Nike Air Max") to evade keyword scraping. Corsearch Zeal 2.0 explicitly targets "disguised infringements even lacking explicit brand identifiers" via deep semantic detection.

This demo implements a CLIP-style bi-encoder retrieval over a live catalog of real products fetched from the DummyJSON open-source API, plus auto-generated "hidden brand" counterfeit twins for each. The query is projected into a shared concept space where it matches listings by visual/semantic DNA — not brand keywords.

How to use: Type a product description (no brand name needed) or click an example. Listings are ranked by semantic similarity. Any listing with high semantic match but no visible brand is auto-flagged ⚠ HIDDEN BRAND — the key catch this system enables.

How it works

Real catalog: products fetched live from DummyJSON (dummyjson.com/products) — men's shoes, watches, women's bags, shoes. For each authentic listing, the demo auto-generates a "hidden brand" counterfeit twin (same image, brand stripped from title + replaced with suspicious descriptors like "premium", "AAA", "factory direct").

Concept embedding space: queries and listings project into a shared 28-dimensional space of visual/semantic concepts (product type, color, material, style cues, logo features, price tier). In production this is a fine-tuned CLIP or SigLIP bi-encoder trained on marketplace image-text pairs.

Cosine similarity retrieval + hidden-brand flag: ranked by cos(q, l). A top result with high semantic similarity but no brand keyword in title is auto-flagged for enforcement review — the highest-value catch for this system.
